To run a production-grade instance of the Worlds Platform, configure the
following infrastructure and services.

## Provision infrastructure

Collect the necessary API keys and credentials for each service:

- **Identity (WorkOS)**: Set up AuthKit and retrieve `WORKOS_API_KEY`,
  `WORKOS_CLIENT_ID`, and `WORKOS_COOKIE_PASSWORD`.
- **Compute (Deno Deploy)**: Generate a personal access token.
- **Storage (Turso)**: Retrieve `TURSO_API_TOKEN` and `TURSO_ORG`.
- **AI models (OpenRouter)**: Retrieve `OPENROUTER_API_KEY`.

<Warning>
  Protect your `WORKOS_COOKIE_PASSWORD` and `TURSO_API_TOKEN` securely. These
  keys guarantee admin or root access to identity management and persistent
  storage.
</Warning>

## Deploy the console

The console is a Next.js application. Deploy it to Vercel, Netlify, or a VPS.

<Tabs>
  <Tab title="Vercel">
1. Import your repository into Vercel.
2. Set the environment variables retrieved in the previous step.
3. Ensure `NEXT_PUBLIC_WORKOS_REDIRECT_URI` matches your deployment callback URL.
  </Tab>
  <Tab title="Netlify">
1. Connect your repository to Netlify.
2. Add the environment variables in your site settings.
3. Ensure `NEXT_PUBLIC_WORKOS_REDIRECT_URI` matches your Netlify URL.
  </Tab>
  <Tab title="VPS">
1. Clone the repository to your server.
2. Create a `.env` file with your environment variables.
3. Build and define a process using PM2 or Docker.
  </Tab>
</Tabs>

## Orchestrate worlds

Once the console is running:

<Steps>
  <Step title="Log in">
    Access your console <Icon icon="gear" /> and log in via AuthKit.
  </Step>
  <Step title="Create an organization">
    Define your organization within the console.
    <Frame>
      <img src="/images/wazoo.svg" alt="Worlds Console dashboard" />
    </Frame>
  </Step>
  <Step title="Automatic provisioning">
    The platform will automatically provision a Turso database, deploy a Deno
    Deploy app, and link them together.
  </Step>
</Steps>

## Cloud embeddings with OpenRouter

Production environments utilize **OpenRouter** for cloud-based embeddings by
default.

<Steps>
  <Step title="Generate API key">
    Visit [OpenRouter](https://openrouter.ai/keys) to get an API key.
  </Step>
  <Step title="Configure console">
    Add `OPENROUTER_API_KEY` to your console environment.
  </Step>
  <Step title="Verify injection">
    The console automatically injects this key into new world instances.
  </Step>
</Steps>
