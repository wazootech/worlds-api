# Routes and API

The Worlds platform utilizes a modular routing system to handle core management
and world-specific data operations.

## Registration

In `server.ts`, the `routes` array lists the paths to available modules. Each
module performs a default export of a function that initializes the router:

```typescript
(serverContext: ServerContext) => Router;
```

## Route handler pattern

Every route handler in the platform follows a consistent execution pattern:

1.  **Path parameters**: Read parameters from `ctx.params`.
2.  **Authentication**: Call `authorizeRequest` to verify the user or agent
    identity.
3.  **Validation**: Parse and validate the request body or query parameters
    using Zod.
4.  **Business logic**: Distribute work to the appropriate service or resolve a
    world-specific database client.
5.  **Response**: Return a success response via `Response.json(record)` or an
    appropriate `ErrorResponse`.
