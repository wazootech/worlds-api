# Testing framework

The Worlds platform utilizes a structured testing framework to ensure the
reliability of its neuro-symbolic components.

## TestContext

Defined in `lib/testing/mod.ts`, the `TestContext` extends the standard
`ServerContext` to provide an isolated environment for automated tests:

- **Isolated main database**: Builds an in-memory main client for each test
  session.
- **Schema initialization**: Automatically initializes the database schema.
- **Service instantiation**: Instantiates all core services using the test
  context.
- **Memory database management**: Uses the `MemoryDatabaseManager` to create
  fresh, initialized in-memory world databases as needed.

## Testing patterns

### Service tests

Service tests verify the logic of individual resource services. These tests
create an in-memory client, initialize the schema, and perform assertions on
CRUD operations and business logic.

### Route tests

Route tests verify the behavior of individual API endpoints:

1.  Initialize the `TestContext`.
2.  Register the route handler with the test context.
3.  Execute requests using `app.fetch()` with a mock `Request` object.
4.  Assert response status codes and JSON response bodies.

### Identity mocking

The `createTestOrganization(context)` helper utility inserts a mock organization
into the test database and returns a valid API key for use in `Authorization`
headers during testing.
