# Resource pattern

The Worlds platform follows a consistent directory structure for implementing
resources and services within the core and world-specific database scopes.

## Directory structure

Each resource resides under `packages/server/lib/database/core/` or
`packages/server/lib/database/world/` and contains the following files:

- **`queries.sql`**: Contains the DDL and CRUD operations.
- **`queries.sql.ts`**: Contains the generated string constants for the SQL
  queries.
- **`schema.ts`**: Defines the Zod schemas for the resource.
- **`service.ts`**: Implements the service class and its methods.
- **`service.test.ts`**: Contains the Deno tests using the in-memory database
  client.

## Service mapping

The following table describes the core services and their respective database
scopes and consumers:

| Resource             | DB scope | Principal consumers                                       |
| :------------------- | :------- | :-------------------------------------------------------- |
| **chunks**           | World    | search route, rdf-patch, testing                          |
| **invites**          | Main     | invites route, testing                                    |
| **logs**             | World    | worlds route (logs), testing                              |
| **organizations**    | Main     | organizations route, testing                              |
| **rate-limits**      | Main     | middleware/rate-limit, init, testing                      |
| **service-accounts** | Main     | organizations route, testing                              |
| **triples**          | World    | rdf-patch, chunks test, testing                           |
| **usage**            | Main     | search route (ChunksService), testing                     |
| **worlds**           | Main     | server.ts, worlds/sparql/search routes, managers, testing |
