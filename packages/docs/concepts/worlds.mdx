A **World** serves as the foundational unit of organization in the ecosystem. It
provides an isolated container for an agent's relationships, history, and facts.

### The analogy: A room

Think of a World as a **Room**. A room provides a scoped environment with unique
objects, rules, and context. Agents can interact with different Worlds to access
specific sets of knowledge, similar to moving between rooms to change
environments.

### Technical equivalent: Isolated RDF dataset

Technically, a World comprises an **Isolated RDF Dataset**. Each World functions
as a self-contained RDF store, often pairing with a search index for
high-performance semantic retrieval. This structure stores facts as
deterministic relationships (symbolic logic) while maintaining searchability via
vector techniques (neural intuition).

---

## Key characteristics

### Portability

Worlds treats memory as a modular component for any "brain" (OpenAI, Anthropic,
etc.). Switching the LLM preserves the World and its associated memory.

### Neuro-symbolic fusion

A World fuses two architectural components:

- **Neural (Intuition):** Uses Vector Search for semantic similarity and natural
  language understanding.
- **Symbolic (Logic):** Uses RDF and SPARQL for verifiable and deterministic
  facts.

### Traceable facts

Worlds eliminates probabilistic "hallucination-prone" RAG by providing a
traceable audit trail for every retrieved fact.
