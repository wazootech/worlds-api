A common point of confusion is the difference between standard
Retrieval-Augmented Generation (RAG) and **Agent Memory**.

- **Standard RAG** answers: _"What is recorded in these documents?"_
- **Agent Memory** answers: _"What is the current, grounded truth about this
  entity?"_

## The Limits of Stateless RAG

In traditional RAG, you chunk and embed text. Retrieval is based on semantic
similarity. This works for static info, but fails for **evolving facts**.

### Scenario: The Evolving Fact

1. **Monday:** "I am working on Project Apollo."
2. **Wednesday:** "I am pausing Apollo to focus on Project Hermes."
3. **Friday:** "What am I working on?"

A traditional RAG system retrieves **both** chunks. The LLM then has to "guess"
based on timestamps or heuristics. As the database grows, retrieval becomes
noisy and conflicting.

## The Worlds Solution: Stateful Memory

Worlds maintains a **Living Knowledge Graph**. Instead of just storing raw text,
Worlds extracts the **meaning** as **Triples** (Subject -> Predicate -> Object).

When you switch projects, Worlds:

1. Identifies the **User** and **Project** items.
2. Updates the `workingOn` relationship, gracefully deprecating the old link.

### Comparison

| Feature       | Traditional RAG     | Worlds (Agent Memory)                   |
| :------------ | :------------------ | :-------------------------------------- |
| **Search**    | Semantic similarity | Hybrid (Semantic + Relational)          |
| **State**     | None (Stateless)    | Stateful (Auto-resolves contradictions) |
| **Inference** | Hallucination-prone | Deterministic Reasoning                 |
| **Structure** | Unstructured Chunks | Knowledge Primitives (Items, Triples)   |
