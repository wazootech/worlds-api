A **Knowledge Graph** powers every World. Unlike traditional databases that
store data in rigid tables, Worlds organizes information in a graph-based
structure. This approach models complex relationships with human-like
flexibility and mathematical precision.

## Statements: The building blocks

A **Statement** represents the fundamental unit of knowledge in a World. Worlds
stores everything an agent knows or remembers as a collection of individual
Statements.

### The analogy: An atom

Think of a Statement as an **Atom**. Atoms are the smallest, indivisible units
of matter. Similarly, a Statement is the smallest unit of knowledge in a World.
While a single Statement represents a simple fact, thousands of Statements
together form a nuanced knowledge graph.

### Technical equivalent: RDF triple

Technically, a Statement is an **RDF Triple**. It follows the standard W3C
structure of **Subject -> Predicate -> Object**.

- **Subject:** The entity (e.g., `User_123`).
- **Predicate:** The relationship or property (e.g., `likes`).
- **Object:** The value or related entity (e.g., `Coffee`).

## Why knowledge graphs matter

### Deterministic truth

Unlike large language model weights, which remain probabilistic and prone to
hallucination, a Knowledge Graph Statement represents a hard fact. When an agent
retrieves a Statement, it accesses a specific, verifiable relationship.

### Malleability

Agents can "fork, merge, and mutate" Knowledge Graphs in real-time. This
capability allows the memory to evolve as the agent learns new information or
corrects assumptions.

### Semantic Searchability

Statements use rigid triples, but the Worlds API indexes them for semantic
search. This indexing allows agents to find the correct "Atom" using natural
language, even when queries do not perfectly match the technical predicate.

## Standards and RDF

Worlds leverages the **Resource Description Framework (RDF)** rather than
reinventing its own standard. RDF comprises a family of World Wide Web
Consortium (W3C) standards originally designed for the Semantic Web.

### Benefits of RDF

- **Interoperability:** Users can export and use data with any RDF-compliant
  tool, ensuring it remains portable.
- **Symbolic Logic:** RDF enables the use of **SPARQL**, a powerful query
  language for complex, deterministic lookups.
- **Data Sovereignty:** By using open standards, Worlds ensuring your knowledge
  remains yours. Worlds stores knowledge at the edge or locally, ensuring
  users—not AI providers—own their personal data.

---

## Abstraction layer

The **Worlds SDK** acts as a "translator" between the developer and the RDF
database. It converts natural language or simple function calls into complex
graph queries. This abstraction makes symbolic logic accessible without
requiring database expertise.
