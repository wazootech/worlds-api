# The Agentic Loop

Worlds is the center of the **Agentic Loop**. An agent doesn't just read from a
World; it participates in a continuous cycle of retrieving context, performing
actions, and **learning** from the results.

## The Three Stages

To build a "Continual Learning" agent, your system should follow this three-step
lifecycle during every interaction:

### 1. Search (Context)

Before the agent sees the user's prompt, the system searches the World for
relevant **Triples**. This "primes" the agent with the exact grounded truth it
needs.

> [!TIP] This stage also pulls the **User Profile**—a holistic digital identity
> formed by every fact ever learned about that specific user.

### 2. Act (Reasoning)

The agent processes the request using the retrieved context. It might execute a
tool or answer a question. During this stage, the agent identifies new facts or
recognizes that an old fact has changed.

### 3. Learn (Materialization)

After the action, the agent **materializes** its new understanding back into the
World.

- **Refining Facts**: Updating the user's current project or preference.
- **Deepening Identity**: Adding a new triple to the persistent user profile.

## Why this Matters

Without the **Learn** stage, an AI has "short-term memory loss." It might be
helpful for one session, but it forgets everything once the window closes. By
completing the loop, you turn a one-off chat into a **Persistent Assistant**
that grows smarter with every turn.
