# Knowledge Primitives

Worlds stores the "reality" it extracts from unstructured data as **RDF
Triples**. To build with Worlds, you need to understand the three fundamental
building blocks of knowledge.

## 1. Items (Entities)

**Items** are the distinct "things" in your World—a person, a piece of code, or
a company.

- **Terminology**: Also referred to as **Resources** or **Named Nodes**.
- **Identification**: Every item is identified by a unique **IRI**
  (Internationalized Resource Identifier).
- **Classes**: Items are categorized by their type (e.g., `User`, `Project`,
  `Task`).

## 2. Properties (Predicates)

**Properties** are the "verbs" or "connectors" that define how Items relate.

- **Examples**: `worksOn`, `managerOf`, `hasPriority`.
- **Ontology**: The set of defined properties forms the "grammar" of your World,
  ensuring the agent uses consistent language (e.g., using `w:prefers` instead
  of guessing between "likes" or "loves").

## 3. Facts (Triples)

A **Fact** is created the moment you connect two Items using a Property. This is
an **RDF Triple**:

1. **Subject**: The item being described (`User:Ethan`).
2. **Predicate**: The relationship (`isWorkingOn`).
3. **Object**: The target item or value (`Project:Worlds-API`).

| Storage Era          | Primitive       | Mental Model                                       |
| :------------------- | :-------------- | :------------------------------------------------- |
| **Traditional Disk** | File / Folder   | Hierarchical: You must know the path to find data. |
| **Worlds**           | **Item / Fact** | Associative: You follow facts to find items.       |

## Why this Architecture Wins

By structuring memory as symbolic facts rather than raw text chunks:

- **Agents Reason**: They can traverse the graph to find hidden relationships
  (e.g., "If Alice manages the team and Bob is on the team, Alice manages Bob").
- **Facts Evolve**: New facts can surgically update or override old ones,
  maintaining a "ground truth" state.
- **Search Resonates**: Retrieval can combine semantic "vibes" with
  deterministic logic.
