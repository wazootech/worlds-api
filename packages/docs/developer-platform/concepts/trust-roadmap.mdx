## The Local-First Manifesto

Worlds is built on the **Local-First Manifesto**. We believe that personal and enterprise knowledge should be:

- **Durable**: Your data persists independently of any cloud service.
- **Private**: You own your markdown and RDF files; zero data tracking by default.
- **Malleable**: Every triple is a surgical truth that you can edit or delete.

## Data Privacy & Residency

We believe in data transparency. This guide explains exactly where your information lives depending on your deployment mode.

### Local Development (Ollama/SQLite)

**No data ever leaves your machine**.

- Metadata is stored in a local `.db` file.
- Embeddings are generated via **Ollama** locally.
- 100% private and air-gapped, featuring zero telemetry.

### Standalone Server (Self-Hosted)

- Metadata is stored in your specified `LIBSQL_URL`.
- Embeddings can remain private if using local providers like Ollama.
- Developers retain 100% ownership and copyright of their knowledge graphs.

### Managed Platform (Console)

- Each world gets its own dedicated **Turso** database and **Deno Deploy** worker.
- You can choose regions during setup to ensure data residency and proximity.

- **Tenant Isolation**: API requests enforce strict routing protocols using tenant headers (e.g., `x-tenant-id`). Memory pools are fully isolated, ensuring an AI only retrieves context from the specific dataset assigned to that user.

> [!IMPORTANT]
> **Navigating Compliance**: As a symbiotic base layer, Worlds must safely bridge open-source, local-first environments with enterprise-SaaS systems. Because Worlds can operate entirely offline, it helps developers satisfy strict **SOC 2 Type I & II**, **GDPR**, and **HIPAA** constraints.
>
> Unlike purely local note-taking tools that cannot sign Business Associate Agreements (BAAs) for custom sync endpoints, Worlds allows enterprises to maintain complete control over where data flows. By enforcing strict network security policies, isolated user profile engines, and secure disposal protocols for electronic media containing sensitive data (PII, ePHI), we ensure your proprietary intelligence never leaks into public model training sets.

## Future Roadmap

Worlds is an evolving neuro-symbolic platform. The following "missing puzzle pieces" are targeted for future development:

### Automated Lifecycle & Syncing

- **Folder Watcher**: CLI mode that monitors local directories and automatically ingests changes.
- **Scheduled Sync**: Automated periodic pulls from external REST APIs.

### Advanced Semantic Tagging

- **Semantic Tagging Spec**: Standardized metadata for categorizing Worlds and Organizations.
- **Inference Rules**: Automatic tagging of triples based on logical patterns.

### Ecosystem Expansion

- **VS Code Extension**: Direct SPARQL editing and World management.
- **Python SDK**: Broadening the toolchain for data science and AI research.
