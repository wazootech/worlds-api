The platform switches between modes based on the presence of specific environment variables.

### Management Console Requirements

These variables are required in `packages/console/.env`.

| Variable                       | Mode       | Purpose                                                                         |
| :----------------------------- | :--------- | :------------------------------------------------------------------------------ |
| `WORKOS_API_KEY`               | Production | Enables Remote WorkOS Management.                                               |
| `WORKOS_CLIENT_ID`             | Production | Required for AuthKit login.                                                     |
| `DENO_DEPLOY_TOKEN`            | Production | Enables remote Deno Deploy orchestration.                                       |
| `TURSO_API_TOKEN`              | Production | Enables remote Turso database provisioning.                                     |
| `TURSO_ORG`                    | Production | The Turso organization slug.                                                    |
| `WORLDS_EMBEDDINGS_DIMENSIONS` | Both       | (Optional) Unified vector dimensions for DB and AI (defaults to `768`).         |
| `OPENROUTER_API_KEY`           | Both       | (Optional) Global fallback for embeddings.                                      |
| `OPENROUTER_MANAGEMENT_KEY`    | Production | (Optional) Enables automatic per-app API key generation via OpenRouter API.     |
| `OPENROUTER_EMBEDDINGS_MODEL`  | Both       | (Optional) Global fallback model (defaults to `openai/text-embedding-3-small`). |
| `OLLAMA_BASE_URL`              | Both       | (Optional) Ollama API URL (defaults to `http://127.0.0.1:11434`).               |
| `OLLAMA_EMBEDDINGS_MODEL`      | Both       | (Optional) Ollama model (defaults to `nomic-embed-text`).                       |
| (None)                         | Local Dev  | If WorkOS keys are missing, the console defaults to file-backed mode.           |

By default, the Worlds Platform uses **Ollama** for local embeddings. This ensures your data never leaves your machine.

> [!TIP]
> You can also use **OpenRouter** for embeddings in local development by adding an `OPENROUTER_API_KEY` to your environment. See [Configuration Reference](/developer-platform/infrastructure/configuration) for details.

### Embedding Provider Selection

The Worlds API picks an embedding provider based on the available environment variables.

| Provider       | Trigger                          | Notes                                                                         |
| :------------- | :------------------------------- | :---------------------------------------------------------------------------- |
| **OpenRouter** | `OPENROUTER_API_KEY` is present. | Recommended for cloud-based AI. Uses per-app keys if automatically generated. |
| **Ollama**     | Default (no cloud key).          | Recommended for local development and private air-gapped hosting.             |

### Worlds API Server Requirements

These are automatically provisioned and injected by the Console during deployment, but required if self-hosting the server manually.

| Variable                       | Purpose                                                                                                                          |
| :----------------------------- | :------------------------------------------------------------------------------------------------------------------------------- |
| `WORLDS_API_KEY`               | (Optional) The **Root Admin Key** for the server. Generated by the provisioner. If omitted, the server runs in unprotected mode. |
| `WORLDS_EMBEDDINGS_DIMENSIONS` | (Optional) Unified vector dimensions for DB and AI (defaults to `768`).                                                          |
| `LIBSQL_URL`                   | Connection string (e.g., `libsql://...` or `file:...`).                                                                          |
| `LIBSQL_AUTH_TOKEN`            | Auth token for remote libSQL/Turso instances.                                                                                    |
| `OPENROUTER_API_KEY`           | (Optional) For high-quality OpenRouter embeddings.                                                                               |
| `OPENROUTER_EMBEDDINGS_MODEL`  | (Optional) OpenRouter model (defaults to `openai/text-embedding-3-small`).                                                       |
| `OLLAMA_BASE_URL`              | (Optional) Ollama API URL (defaults to `http://127.0.0.1:11434`).                                                                |
| `OLLAMA_EMBEDDINGS_MODEL`      | (Optional) Ollama model (defaults to `nomic-embed-text`).                                                                        |
