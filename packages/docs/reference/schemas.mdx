The Worlds Platform utilizes a dual-scope database architecture to separate
management logic from world-specific data.

## DatabaseManager

The `DatabaseManager` interface provides a common API for world database
lifecycle management:

- `create(id: string)`: Creates a new world database and initializes its schema.
- `get(id: string)`: Retrieves a `ManagedDatabase` instance for the specified
  world.
- `delete(id: string)`: Removes the database resources for the specified world.

## Generation

The platform uses a structured approach to SQL management and type safety:

- **Queries**: Live in `.sql` files near their respective resources.
- **Code generation**: An internal utility embeds SQL queries into TypeScript
  constants at build time `.sql.ts` to ensure runtime availability.
- **DDL execution**: The system orchestrates table and index creation during
  initialization to ensure correct execution order.
