The Worlds API Server is built on Deno for secure, edge-ready execution with
web-standard APIs.

## Request lifecycle

Every request to the Worlds API passes through a standardized lifecycle to
ensure security, consistency, and efficient graph operations.

<Steps>
  <Step title="Routing">
    The server matches the incoming request URL and method to the appropriate handler.
  </Step>

<Step title="Authorization">
  For secured endpoints, the server verifies the `Authorization: Bearer` token.
  See the [Authentication guide](/overview/getting-started/authentication) for
  details.
</Step>

<Step title="Validation">
  Input data (body and query parameters) is validated against strict schemas to
  ensure type-safety and data integrity.
</Step>

<Step title="Execution">
  The server performs the requested operation. For world-specific requests, it
  automatically resolves and interacts with the correct knowledge graph
  instance.
</Step>

  <Step title="Response">
    The system returns a serialized JSON response or a standardized error message.
  </Step>
</Steps>
