# Deployment

Worlds is designed for polymorphic deployment. You can run the entire stack
locally for development and scale to a managed production environment with the
same codebase.

## 1. Local Development (Small Web)

Run the entire ecosystem—Management Console and World Servers—on your local
machine.

```bash
# Clone the repository
git clone https://github.com/wazootech/worlds.git
cd worlds/packages/console

# Install and run
npm install
npm run dev
```

The console will automatically spawn local World Server processes using
**SQLite** for storage and **Ollama** for local embeddings.

## 2. Managed Platform (The Console)

Our managed platform provides a multi-tenant Control Plane that handles identity
(WorkOS) and automated provisioning of World instances.

- **Control Plane**: Manages organizations and API keys.
- **Data Plane**: Each organization receives isolated compute workers and
  **Turso (libSQL)** databases.

## 3. Standalone Self-Hosting

You can host a single World Server instance without the Management Console in
any Deno-compatible environment.

```bash
# Set required env vars
export LIBSQL_URL="file:worlds.db"
export WORLDS_API_KEY="your-secret-key"

# Start the server
deno task start
```
