Worlds supports multiple strategies for ingesting and continuously updating data
within an isolated world memory container.

<Note>
  You'll need an [API Key](/reference) and the [Worlds SDK](/quickstart) installed to run these examples.
</Note>

<CodeGroup>

```typescript TypeScript
import { WorldsClient } from "@wazoo/worlds-sdk";

const client = new WorldsClient({ token: process.env.WORLDS_API_KEY });

// Insert a single triple fact into the world
await client.worlds.triples.insert("world-id", {
  subject: "ex:User123",
  predicate: "rdf:type",
  object: "ex:Contributor",
});
```

```python Python
from worlds import WorldsClient
import os

client = WorldsClient(token=os.environ.get("WORLDS_API_KEY"))

# Insert a single triple fact into the world
client.worlds.triples.insert("world-id",
    subject="ex:User123",
    predicate="rdf:type",
    object="ex:Contributor"
)
```

```bash HTTP API
curl -X POST "https://api.wazoo.dev/v1/worlds/world-id/triples" \
  -H "Authorization: Bearer $WORLDS_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "subject": "ex:User123",
    "predicate": "rdf:type",
    "object": "ex:Contributor"
  }'
```

</CodeGroup>

## Core methods

| Method           | Description                                        |
| :--------------- | :------------------------------------------------- |
| `import`         | Ingest bulk RDF files or text representations.     |
| `triples.insert` | Synchronously insert discrete items or properties. |
| `triples.delete` | Remove facts from the stateful world memory.       |

## Deep dive

### Ingestion strategies

#### One-time import

Use the CLI or SDK to perform a bulk import of existing RDF data. This is ideal
for bootstrapping a new world with a base ontology or research dataset.

```bash
# Import a Turtle file into a world
worlds import --world <world-id> --file ./ontology.ttl
```

#### Direct SDK ingestion

For real-time applications, use the `worlds-sdk` to insert individual triples or
chunks as they are generated by your agentic loops.

### Data synchronization

To keep your world memory in sync with external sources, utilize our
forward-sync proxy.

1.  **Map external events**: Capture changes in your source system.
2.  **Translate to patches**: Convert changes into RDF patch operations.
3.  **Apply patches**: Use the `rdf-patch` service to update the world graph
    atomically.

<Tip>
  Always skolemize blank nodes before ingestion to ensure deterministic triple
  IDs.
</Tip>
