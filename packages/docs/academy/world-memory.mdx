A **World** functions as the foundational unit of organization in the ecosystem.
It provides an isolated container for an agent's relationships, history, and
facts.

## Key takeaways

- A World serves as an isolated container for agent relationships and facts.
- Each World comprises an isolated RDF dataset paired with a search index.
- World memory remains portable across different large language models.

## Technical foundations: Isolated RDF dataset

Technically, a World comprises an **isolated RDF dataset**. Each World functions
as a self-contained RDF store. This structure pairs with a search index for
high-performance semantic retrieval. This architecture stores facts as
deterministic relationships (symbolic logic) while maintaining searchability via
vector techniques (neural intuition).

## Key characteristics

### Portability

Worlds treats memory as a modular component for any "brain" (OpenAI, Anthropic,
or other providers). Switching the LLM preserves the World and its associated
memory.

### Neuro-symbolic fusion

A World fuses two architectural components:

- **Neural (intuition):** This component uses vector search for semantic
  similarity and natural language understanding.
- **Symbolic (logic):** This component uses RDF and SPARQL for verifiable and
  deterministic facts.

### Traceable facts

Worlds eliminates probabilistic, hallucination-prone retrieval. It provides a
traceable audit trail for every retrieved fact.
